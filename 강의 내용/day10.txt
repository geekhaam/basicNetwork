※ Routing

   - 데이터를 최적의 경로를 선택하여 목적지까지 이송하는 모든 절차
   - IP Protocol이 동작하는 Router, L3-Switch(Multi-Layer Switch), Computer
   - Routing을 위해 장비가 알아야 하는 것들은 다음과 같다.
     1. Source(출발지)/Destination(목적지) 주소
     2. Inbound/Outbound 인터페이스 형태
     3. 가능성 있는 모든 경로들에 대한 정보
     4. 지속적인 네트워크 상태 확인 및 유지

   - Router가 Routing을 위한 정보를 학습하는 방법
     1) Connected : 직접적으로 네트워크에 연결된 정보로 장비의 각 인터페이스에 설정한 네트워크 정보
     2) Static : 관리자가 입력한 네트워크 정보로 해당 장비가 특정 목적지로 데이터를 이송할 수 있게 입력한 정보
     3) Dynamic : 장비에 설정한 Routing Protocol에 의해 자동적으로 교환된 정보(RIP, EIGRP, OSPF, ...)
     4) Redistribute : 재분배, 서로 다른 형태로 학습한 정보를 교환하여 학습할 수 있게 한다.

   - 학습한 정보들은 Routing Table에 기록이 되며, Routing Table에 기록된 내용을 바탕으로 데이터를 이송한다.
     Routing Table에 기록된 정보는 최적 경로에 해당하는 정보만이 기록된다.

	학습 방법		목적지 네트워크		출구 정보(인터페이스 또는 Next-Hop)
	connected	192.168.1.0/24		eth 0/0
	static		192.168.2.0/24		eth 0/1
	dynamic		192.168.3.0/24		eth 0/0
	redistribute	192.168.4.0/24		eth 0/1

   - Static / Dynamic 학습 방식에 따른 차이
	Static	 : 관리자가 집접 수동으로 필요한 정보를 입력
		   추가적으로 발생한 정보에 대해서도 다시 입력이 필요

	  	   네트워크 변화에 대해 Router가 자동 감지를 못함
		   관리자가 직접 네트워크 변화에 대처
		 
		   한 번 입력한 정보는 계속 유지되어 사용
		   추가 학습을 위한 프로세스 자원이 필요 없음
		   네트워크 변화를 감지하기 위한 추가 자원이 필요 없음 -> 최소한의 프로세스 자원으로 통신 가능

	Dynamic	 : Routing Protocol을 통해 자동으로 정보 학습
		   추가적으로 발생한 정보에 대해서도 자동으로 학습

		   네트워크 변화에 대해 Router가 자동적으로 감지함
		   관리자가 직접 네트워크 변화에 대처할 필요 없음

  		   학습된 정보는 네트워크 변화에 따라 추가/수정/삭제
		   추가 학습을 위한 프로세스 자원이 많이 필요
		   네트워크 변화를 감지하기 위한 추가 자원이 필요

    - Static Route 구성
	라우터가 데이터를 목적지에 이송할 수 있도록 목적징 정보와 출구 정보를 알려주는 방식으로 구성한다.
	Router(config)# ip route <network> <mask> {local-interface | next-hop-ip}
	- network : 목적지 네트워크 주소 정보
	- mask : 목적지 네트워크 주소의 마스크 정보
	- local-interface : 현재 설정하고 있는 라우터에서 목적지로 데이터를 이송시켜야 하는 출구 인터페이스
	- next-hop-ip : 현재 설정하고 있는 라우터에서 목적지로 데이터를 이송시키기 위한 인접 라우터의 주소


 192.168.1.0/24		192.168.2.0/24		192.168.3.0/24		192.168.4.0/24
----------------- R1 -------------------- R2 -------------------- R3 --------------------
	    fa0/0    fa0/1	    fa0/1    fa0/0          fa0/0    fa0/1
	       .1    .1		       .2    .2                .1    .1

 R1에서 R3 192.168.4.0/24 네트워크까지 데이터를 이송시킬 수 있게 하기 위한 Static 구성
 R1(config)# ip route 192.168.4.0 255.255.255.0 fa0/1
 또는
 R1(config)# ip route 192.168.4.0 255.255.255.0 192.168.2.2

 R1의 Routing Table
	학습 방법		목적지 네트워크		출구 인터페이스
	connected	192.168.1.0/24		fa0/0
	connected	192.168.2.0/24		fa0/1
	static		192.168.4.0/24		fa0/1 또는 192.168.2.2

 R2에서 R3의 192.168.4.0/24 네트워크까지 데이터를 이송시킬 수 있게 하기 위한 Static 구성
 R2(config)# ip route 192.168.4.0 255.255.255.0 fa0/0
 R2(config)# ip rotue 192.168.4.0 255.255.255.0 192.168.3.1
